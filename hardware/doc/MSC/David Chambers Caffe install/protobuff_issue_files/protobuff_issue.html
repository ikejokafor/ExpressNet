This message is from dnn_modern (opencv_contrib). It requires system protobuf with available compiler (OpenCV has "3rdparty" protobuf runtime library without protoc compiler).

Unfortunately, opencv_dnn_modern module has strong conflict with opencv_dnn.
This conflict can't be resolved easily because both tries to use custom "caffe" schemes.

You can try to disable opencv_dnn (-DBUILD_opencv_dnn=OFF), OpenCV's builtin protobuf (-DBUILD_PROTOBUF=OFF) and re-run CMake from clean build directory.

alalek commented on Nov 20, 2017

OpenCV DNN module is "opencv_dnn" (not "opencv_dnn_modern").

"opencv_dnn_modern" is disabled in default build of OpenCV.

To force custom pre-built protobuf library you need to pass these CMake variables:

  • -DBUILD_PROTOBUF=OFF -DPROTOBUF_UPDATE_FILES=ON
  • -DProtobuf_LIBRARY=...
  • -DProtobuf_INCLUDE_DIR=...
  • -DProtobuf_PROTOC_EXECUTABLE=...

(should work fine after merging of #10092)
